# 欧派 AI 灵感家 MVP

本项目严格依据《欧派AI灵感家_MVP产品需求文档_v1.0》实现。当前主链路覆盖P02需求输入至P07交付看板，未接入或宣称任何欧派真实客户、订单、价格、SKU或生产系统数据。

## 已实现范围

- 城市、户型、面积、预算上下限、家庭结构、入住时间、特殊需求与10—500字自由描述
- 匿名客户创建、编辑、切换和删除；林女士固定案例可一键重置并演示
- 每位客户独立保存视觉事件、推荐预算、所选案例和风险处置
- 浏览器 `localStorage` 保存与刷新恢复，不依赖数据库或API
- 七轮视觉选择：整体氛围、装修风格、冷暖色、材质、收纳方式、灯光、个性元素
- 五种操作：喜欢、不喜欢、都可以、跳过、撤销上一步
- 客户画像实时更新，并显示“客户明确选择 / 系统推断”和置信度
- 固定冲突场景：大量开放格与易清洁、儿童安全、视觉整洁冲突
- 折中建议：20% 开放展示＋80% 封闭收纳
- 图片卡片与放大图均由前端叠加“AI生成示意图”标签
- 读取60条模拟案例，先执行合规/硬约束过滤，再生成三种不重复推荐
- 七项固定权重基础评分：风格25%、预算20%、户型面积15%、家庭结构15%、功能10%、成交表现10%、满意度5%
- 每套推荐展示3条字段证据、1条风险/取舍、七项分数、权重和算法版本
- 支持将预算25万元切换为20万元或30万元并实时重新排序
- 选择任一推荐方案后建立模拟项目单，展示量尺至验收八阶段
- 基于240条模拟SKU逐行核对报价与交期，固定触发报价漏项和跨品类交期不一致
- 两项风险均支持模拟处置，并记录处置日志、更新阶段与模拟进度
- 1366×768 桌面端及 390px 移动端响应式布局
- Express MVC 接口边界、Zod 校验和 Prisma PostgreSQL 数据模型

## 本地运行

要求 Node.js `>=22.13.0`。

```powershell
npm.cmd install
npm.cmd run dev
```

默认演示不需要启动数据库或API。如需单独验证预留的Express API：

```powershell
Copy-Item .env.example .env
npm.cmd run api:dev
```

当前演示站默认使用 `http://localhost:3000`，示例环境已将该地址配置为 API 允许的跨域来源；如果前端端口发生变化，请同步修改 `CORS_ORIGIN`。

推荐接口为 `POST /api/recommendations`，请求体包含视觉选择事件和 `budgetMax`。前端保留本地规则计算和浏览器持久化，因此数据库、外部AI或接口不可用时仍可完成MVP核心链路。

交付接口为 `POST /api/delivery/projects`，请求体包含 `caseId` 和已处理风险编号。所有项目日期、SKU、价格、进度和风险事件均为本地模拟。

## 模拟边界

- 页面中的“林女士”是v1.0文档规定的固定演示客户；其他客户称呼应使用匿名编号或脱敏称呼。
- 多客户数据只存在当前浏览器的 `localStorage`，不同电脑之间不会共享，也不会随Git仓库上传。
- 比赛版不设置手机号、身份证或详细地址字段。
- 当前 30 张图片全部为 AI 生成或基于公开参考再创作的演示图，不能作为真实成交或交付证明。
- 推荐、冲突与置信度均为可解释规则演示，不代表真实模型训练结论。
